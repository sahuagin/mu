## Surprises

1. **`jj` working-copy semantics surprise.** The session inherited a "dirty" working copy that was actually the experiment-2 briefing landing as the @ revision. Because jj's working copy IS the current revision (not a separate uncommitted-changes area), my mu-ktq files initially got mixed into the experiment-2 commit. Resolved cleanly with `JJ_EDITOR=true jj split .beads/issues.jsonl MORNING_2026-05-13b.md`, which extracted the experiment-2 files into the parent revision and left the remaining changes in a new child for the mu-ktq commit. Tool-conventions.md already flagged `jj split` as the load-bearing primitive for post-hoc concern separation; this session was a clean test of that guidance.

2. **`br close -m` vs `-r`.** The br CLI uses `-r/--reason` for close, not `-m`. Tool-conventions.md doesn't mention this; suggested update below.

## Skill / convention updates suggested

1. Add to `~/.claude-personal/skills/goal-protocol/references/tool-conventions.md` under the `br` section: `br close` uses `-r/--reason`, NOT `-m/--message`. Using `-m` triggers `unexpected argument '-m'` with no helpful inference.

2. Add to the same section: when working in a jj repo where the operator was the previous-session author and the current working-copy revision IS a previous-session commit, **always check `jj st` and `jj log -r '@'` before writing files**. If @ is a meaningful prior commit, do `jj new` first to create a fresh child revision; otherwise edits accumulate into the prior commit. This session caught the mix and split it post-hoc, which is recoverable but extra work. Cleaner: `jj new` immediately after pre-flight reading if @ already has a description.

3. The "Stretch X = no commits" wording in goal text is mildly ambiguous in a jj workspace where br/file edits MUST land somewhere. Recommended re-wording for future experiments: "Stretch A (br create) and Stretch C (memory write): no SEPARATE commits — fold into the primary bead's commit." That matches what the discipline actually wants and is unambiguous.
